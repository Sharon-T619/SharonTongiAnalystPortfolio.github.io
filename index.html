Heart Failure Prediction using ML

Using Jupyter Notebook, the heart dataset was analyzed and processed before the ML models were built and tested using the training data and test data, respectively. The project relied on various libraries such as Numpy, Pandas, Scipy.stats, OS, Seaborn, Time, and Matplotlib. For the purpose of modeling, I deployed six models from Sklearn libraries: RandomForest, KNeighbors, LogisticRegression, SVC, GradientBoosting, and GaussianNB. After conducting the preliminary data analysis (dataset information, checking data types, checking the presence of null values, and finding duplicate values), I went ahead and conducted variable analysis using plots, both histograms and KDE plots, then checking data skewness and finding outliers in the set using combined plots (boxplots and stripplots). Then I ran univariate analysis on the categorical data using barplots, followed by bivariate analysis using KDE plots, heatmaps, and count plots to explore the relationships between two variables. Before building the models, I performed data preprocessing (label encoding, standardization, train_test_split [70% train, 30% test]) because clean data is needed to build an accurate model. Using the 6 model, I was able to train and predict results using a 10-fold CV and confusion matrix. The hyperparameters were optimized using GridSearchCV before the model evaluation was done. A confusion matrix of precision, recall, accuracy, and F1-score was generated. To finalize the project, model evaluation using the ROC and AUC from roc_curve and sklearn metrics was used, plotting the ROC curve and printing the AUC. From the outputs, RF, KNN, and SVC performed well in predicting heart failure.

Stroke Prediction using Random Forest
in R

Using libraries such as tidyverse, caret, gridExtra, and randomForest, I was able to analyze the stroke data, train, and test the performance of an Rf model. The project involved changing data types (converting character columns into factors), generating a statistical distribution of the dataset, and dropping a single row from the dataset in the gender column that had 'Other'. From the dataset, approximately 5% of the total entries were missing in the BMI column, so I used the mean values of the column to replace them. After the missing values and data types had been configured properly, graphs for each variable were generated to gain insights into the dataset using ggplots. The relationship between the variables and the target variable [stroke possibility] was then explored. For model building and prediction, the dataset was split into 70% for the train set and 30% for the test set. RandomForest was then used for the data modeling, giving an Out-of-Bag estimate error of 7.13%,  number of trees of = 500 and variables at each split = 3. A confusion matrix was used to see how the classification model performed on the test data. In conclusion, the model was well trained on the training data set.

Salary prediction
in Jupyter

Using Pandas, Numpy, Matplotlib, and Seaborn, I was able to do a preliminary analysis of the dataset and deal with the most obvious issues with the data first (nulls, duplicates, and unnecessary columns). The column Job Titles had 191 unique values, which was difficult to analyze, so I had to categorize them into groups of similar job domains. The education level was also represented in two different ways, so I grouped the values into four distinct groups. Followed by EDA, looking to understand the distribution of the data using pie charts, histograms, and countplots. To explore the relations between independent variables and the target variable (salary), I used scatter plots, box plots, violin plots, and bar plots. Data processing was then performed before prediction models (Decision Tree Regressor and Random Forest Regressor) were built. GridSearchCV was used for the purpose of hypertuning the models; the models were then rebuilt using the best parameters and used to predict the employee salary using the test set. A model evaluation was conducted where actual and predicted values were visualized using a distplot. The metrics used for this project included r2_score, mean_squared_error, and mean_absolute_error from sklearn metrics. I concluded that employee salaries are dependent on years of experience, job title, and education level. The RF regressor performed better than the DT regressor, but not by much.
